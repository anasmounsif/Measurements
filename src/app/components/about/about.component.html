Project overview

The project focuses on displaying vehicles along with their detailed measurements and includes a simple mock login form. Upon successful login, users can view vehicles fetched from the server, each associated with specific measurements.

By selecting a vehicle, users are directed to a details view where the component requests the backend for measurements related to that vehicle based on its ID in the route.

Here is the link to view the mock on figma.

Challenges and Solutions

A simulated login flow was introduced for this project. The token, generated client-side, is stored in localStorage to maintain application state. Conversely, the logout process removes this data from localStorage. The application gracefully handles the scenario where a searched vehicle is unavailable; an alert is triggered, and the user is redirected to the main view displaying vehicles.

Various Angular features were utilized for code implementation, such as attributive directives for customizing headlines, route guards to restrict data access based on user authentication status, and resolvers to fetch necessary data for a component before navigation. Services were used for backend communication, authentication utilities, and localStorage persistence.

Future Directions

Unfortunately, json-server does not support token management for authentication. Ideally, a middleware would be implemented to generate tokens for the client, enabling further operations, followed by the implementation of an interceptor to decorate HTTP calls with the token.

The data used are simple and limited; in a production scenario, data volume will significantly increase. Potential approaches to manage large datasets include enhancing grids to accommodate all data, implementing server-side and client-side pagination, or an infinite scrolling mechanism to avoid overloading with a single getAll call. For real-time data requirements, websockets could be a viable solution.

Furthermore, they can be implemented in facilities to allow the user to understand the data he sees, such as modals with details, calculations and predictions on the progress of measurements
